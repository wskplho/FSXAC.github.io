New project!

This project is not included in the project book, but the idea is simple:

Have a number counter that can be manipulated using a button, and output that number to a LCD. I also wanted another button that switches display mode. For example, one mode would be to display a static number count, another mode would flash through different messages.

I think this project would be more of a challenging task at the beginning since I just done the LED projects. Doing stuff with a LCD is a huge leap.

13:21 2014-11-21

I followed the tutorial on Arduino's official website on how to wire up an LCD. The wiring itself was pretty straight forward, but slightly more confusing than ordinary LEDs. Single coloured LEDs have two pins they needed to be connected to, one is power, the other is ground. However, on the LCD, it's not as simple.

On a LCD, like anything, it needs two pins that connect to power and ground. But an additional set of the power and ground pins is required if I want it to be backlit. I can add a potentiometer where the back-light connection is so that I can control the brightness. A display contrast pin from the LCD is required to connect to power. Without it, the crystals in the screen will not rotate, therefore will not produce an image. Adding a potentiometer would change the contrast. For the pins that require more understanding of computer hardware: There is the RS (register select) pin that controls where in the LCD's memory the data is writing to. There is the R/W (pin) which selects whether if the LCD is in read mode or write mode. And finally, there is E (enable) pin that enables writing to the registers. I don't exactly know how to do cool stuff with these pins, they are essential to make the LCD work, so I'm not going to mess with them. Just connect these pins in the right place and it'll be fine.

Finally, we have the data pins on the LCD. There are 8 data pins, but minimum of only 4 is required to make the LCD display something.
